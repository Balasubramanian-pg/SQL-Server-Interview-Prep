# 44. SQL Server on Linux: Architecture Differences

Canonical documentation for 44. SQL Server on Linux: Architecture Differences. This document defines the conceptual model, terminology, and standard usage patterns.

> [!NOTE]
> This documentation is implementation-agnostic and intended to serve as a stable reference.

## 1. Purpose and Problem Space
Describe why 44. SQL Server on Linux: Architecture Differences exists and the class of problems it addresses.
SQL Server on Linux: Architecture Differences exists to provide a comprehensive understanding of the architectural variations between SQL Server on Windows and SQL Server on Linux. This topic addresses the class of problems related to deploying, managing, and optimizing SQL Server on Linux, including differences in file systems, networking, security, and performance optimization.

## 2. Conceptual Overview
Provide a high-level mental model of the topic.
The conceptual overview of SQL Server on Linux: Architecture Differences involves understanding the key components and interactions between SQL Server, the Linux operating system, and the underlying hardware. This includes the SQL Server engine, storage subsystem, network configuration, and security features.

## 3. Terminology and Definitions
| Term | Definition |
|------|------------|
| SQL Server on Linux | The version of SQL Server designed to run on Linux operating systems |
| Linux File System | A file system used by Linux operating systems, such as ext4 or XFS |
| Windows File System | A file system used by Windows operating systems, such as NTFS |
| Network Configuration | The setup and configuration of network interfaces, protocols, and settings |
| Security Features | Mechanisms and tools used to protect SQL Server and its data, such as authentication and access control |

## 4. Core Concepts
Explain the fundamental ideas that form the basis of this topic.
The core concepts of SQL Server on Linux: Architecture Differences include:
* File system differences: Understanding the differences between Linux and Windows file systems, including file system layout, permissions, and access control.
* Networking differences: Recognizing the differences in network configuration and protocols between Linux and Windows.
* Security differences: Identifying the security features and mechanisms available on Linux and Windows, including authentication, access control, and encryption.
* Performance optimization: Understanding the performance characteristics and optimization techniques for SQL Server on Linux.

## 5. Standard Model
Describe the generally accepted or recommended model.
The standard model for SQL Server on Linux involves:
* Using a supported Linux distribution, such as Ubuntu or Red Hat Enterprise Linux.
* Configuring the Linux file system to optimize performance and security.
* Setting up network configuration to ensure reliable and secure communication.
* Implementing security features, such as authentication and access control, to protect SQL Server and its data.
* Monitoring and optimizing performance to ensure optimal database operation.

## 6. Common Patterns
Document recurring, accepted patterns.
Common patterns for SQL Server on Linux include:
* Using a containerization platform, such as Docker, to deploy and manage SQL Server.
* Implementing a high-availability solution, such as Always On Availability Groups, to ensure database availability.
* Using a cloud-based platform, such as Azure or AWS, to deploy and manage SQL Server on Linux.
* Utilizing a monitoring and management tool, such as SQL Server Management Studio, to monitor and optimize database performance.

## 7. Anti-Patterns
Describe common but discouraged practices.
Anti-patterns for SQL Server on Linux include:
* Using an unsupported Linux distribution or version.
* Configuring the Linux file system in a way that compromises performance or security.
* Failing to implement adequate security measures, such as authentication and access control.
* Overlooking performance optimization techniques, such as indexing and query optimization.
